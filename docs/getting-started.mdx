Agent Kanban keeps you organised while running multiple coding agents in parallel by streamlining how you plan and review their work.

## 1. Launch Agent Kanban

Start Agent Kanban with a single command — no sign-in or account required:

```bash
npx vibe-kanban
```

This downloads the latest release, starts the backend server, and opens the UI in your browser automatically. All data is stored locally in a SQLite database on your machine.

<Info>
Running on a remote server or want Agent Kanban to start automatically on boot? See the [Daemon Mode](/self-hosting/daemon-mode) guide.
</Info>

## 2. Confirm your preferences

The first time you run Agent Kanban, you'll be asked to set your preferred:

- Coding agent
- IDE
- Notification preferences

<Info>
These preferences can be changed at any time from the settings dialog.
</Info>

## 3. Navigate the kanban board

After setup, you land on your first project's kanban board. You can create additional projects at any time from the projects page.

<Frame>
  <img src="/images/onboarding-projects.png" alt="The projects page" />
</Frame>

**Navigating the kanban board:**

1. The app bar, used for navigating between projects, the workspaces page, and user settings
2. Issues appear as cards on the kanban board
3. The **New issue** button for creating issues
4. The right-hand panel where details for the currently selected or draft issue are shown

## 4. Create an issue

**Issues** are a core concept of Agent Kanban — they represent a bug, feature, or piece of work to be done. At a minimum, issues consist of a title and description, but you can also add priorities, tags, and even connect issues together with parent/child relationships.

<Frame>
  <img src="/images/onboarding-create-issue.png" alt="Create issue" />
</Frame>

When you've filled out the details press **Create issue**.

## 5. Create a workspace

**Workspaces** are another core concept of Agent Kanban — they represent a space to work on an issue with a coding agent. When you create a workspace, Agent Kanban automatically creates git worktrees for your selected repositories and launches your coding agent.

<Frame>
  <img src="/images/onboarding-create-workspace.png" alt="The create workspace button" />
</Frame>

To create a workspace, make sure the issue you created is selected and click the **Create** button in the workspaces panel.

<Frame>
  <img src="/images/onboarding-workspaces-repo.png" alt="Workspaces repos" />
</Frame>

When you create a workspace, you'll need to specify the repositories you'd like to work on and the branches to base the git worktrees on.

<Frame>
  <img src="/images/onboarding-workspaces-prompt.png" alt="Workspaces prompt" />
</Frame>

You'll also need to specify your desired coding agent configuration (e.g. model, effort level, plan mode).

<Frame>
  <img src="/images/onboarding-workspaces-logs.png" alt="Workspaces logs" />
</Frame>

Upon creation, the coding agent immediately begins executing with the given prompt.

<Info>
  You can connect multiple workspaces to an issue — useful for working on larger features and running multiple coding agents in parallel.

  Workspaces don't *have* to be connected to an issue, which is useful for quick actions like asking questions about a codebase.
</Info>

## 6. Reviewing a workspace

So far you've been viewing the workspace side-by-side with the kanban board. If you want more room to review the code changes or test them in a browser, open the workspace in the **workspaces view**.

<Frame>
  <img src="/images/onboarding-workspaces-open.png" alt="Workspaces open" />
</Frame>

To access the workspaces view, click the **Open Workspace** button.

<Frame>
  <img src="/images/onboarding-workspaces-page.png" alt="Workspaces page" />
</Frame>

Depending on what you'd like to do, you can view code changes or preview changes to websites in a browser using the floating navigation buttons (3).

You can also navigate back to either your project (1) or the parent issue (2).

## 7. Merging a workspace

When you're ready to merge the changes in a workspace, you can either open a GitHub pull request or merge the workspace branch locally.

<Frame>
  <img src="/images/onboarding-workspaces-merge.png" alt="Workspaces merge" />
</Frame>
